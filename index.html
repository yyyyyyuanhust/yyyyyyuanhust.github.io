Virtual DOM

DOM VS 虚拟DOM

传统DOM

传统DOM更新:
真实页面对应的DOM树在每次页面更新时都需要手动操作DOM进行更新
JS每次修改DOM DOM树都会更新一次(无缓冲)
即做一次如下操作
1、HTML Parser生成DOM Tree
2、CSS Parser生成样式表
3、DOM Tree关联样式表 生成 Render Tree
4、页面布局Layout
5、页面绘制Paint
产生大量的资源浪费 效率极低

虚拟DOM

是一种前端视图展示和更新策略
虚拟DOM是一个轻量级JavaScript对象
类似于

let vNode = {
    tag:'div',
    attributes:{
        id:'xxx',
        class:'xxx'
    },
    children:[
        ...
    ]
}

JS需要修改dom时 操作的是js变量v-dom 效率更高

传统DOM更新:
JS n次操作v-dom后 v-dom可以一次性反映(Diff算法)到dom中(缩减操作次数 避免性能浪费)
优点:更新html效率高,使用vdom更新html真实DOM时首先会对比上一次的状态 (Diff算法)只更新发生改变的dom

HTTPS

HTTPS

https = http + ssl (secure socket layer)
端口443

解决问题

http的明文传输可能被拦截 引起重要信息泄露

加密手段

对称加密

常用算法:DES 3DES AES
原理:使用密钥s加密后必须使用s解密
优点:计算量小,加密解密速度较快,适合加密较大的数据
缺点:密钥的传输容易泄露 一个用户对应一个密钥 服务器管理麻烦

非对称加密

常用算法:RSA
原理:有一个公钥(public key)和一个私钥(private key),由加密算法生成.公钥加密只能私钥解密,私钥加密只能公钥解密
服务器保存了公钥和私钥,将公钥传递给客户端,客户端使用公钥加密后传递数据发送到服务器,服务器使用私钥解密
优点:加密解密使用不用钥匙,可以传输公钥,数据传输安全
缺点:计算量大,加密解密速度较慢

https加密过程

https结合了对称解密和非对称加密的优势,对数据进行加密和解密步骤如下
1、客户端对服务器发起https请求
2、服务器生成一个公钥和一个私钥,将SSL数字证书(包含公钥、服务器相关信息)发送给客户端
3、客户端解析SSL数字证书拿到公钥,验证服务器信息,客户端生成一个随机数使用公钥加密
4、客户端向服务器传输公钥加密后的随机数
5、服务器获取这个公钥加密后的随机数后用私钥解密
6、客户端使用随机数作为密钥对数据进行对称加密(此时服务器也知道密钥因此无需传输密钥)后传输
7、服务器获取密钥加密的数据 使用随机数密钥解密数据

CA证书

域名型(DVSSL)
企业型(OVSSL) 百度
增强型(EVSSL) 苹果

分析

最长摇摆子序列

最长摇摆子序列
给定数组arr 元素为0-9的数字
对于第i位k 若有k>前一位且k>后一位 则这三位 构成一个摇摆序列 例如121
求arr最长摇摆子序列的长度
例如arr=[1,2,3,4,3,2,3,2]
结果是5 其中一个最长摇摆子序列为14232

分析

使用res保存结果 初始值1
什么时候res的值增加?
寻找贪心规律 若从第i位开始向后扫描 一直有后一位大于前一位(记为状态’UP‘) 或 小于前一位(记为状态’DOWN‘)
则不发生摇摆
一旦状态发生了改变则发生了一次摇摆 结果需要增加1
初始状态记为’MID‘ 若后一位不等于arr[0]则状态改变 结果加1 否则代表不变 如’111’长度仍为1
