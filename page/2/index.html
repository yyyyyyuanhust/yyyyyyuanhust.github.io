删除K位后的最小数字

删除K位后的最小数字
给定一个数字number 一个数字k
求number删除k位后的最小值
例如
输入 num=34521 k=3
输出 res=21

分析

举例子
k=1 number=321 res=21
k=1 number=123 res=12
k=1 number=231 res=21
k=1 number=213 res=21
k=2 number=12345 res=123
k=2 number=23412 res=2312=>212
k=2 number=21324 res=1324=>124
则k=1时,需要删除哪一位?
显然从左向右 若第i位<第i-1位 应删除第i-1位
若不存在第i位<第i-1位 从末尾删除
可以使用栈的思想完成
元素依次入栈
若当前元素比前一位小 前一位出栈k– 只要栈中有元素 继续比较(条件是k>0) 直到栈底<当前元素
将当前元素push入栈
扫描结束若k>0 则出栈k次

分糖果

分糖果
已知一些孩子和一些糖果
每个孩子有需求因子g 每个糖果有大小s
若糖果大小大于等于需求因子 即g<=s 此糖果满足改孩子
求最多满足孩子的数量
例
输入g=[5,10,2,9,15,9] s=[6,1,20,3,8]
输出3

分析

贪心思想:对排好序的g,s s[j]优先满足g[i] 其中i,j足够小
对于例子 先排序:
g=[2,5,9,9,10,15]
s=[1,3,6,8,20]
糖果s向右扫描 s[i]>g[j]时候 说明i之前的糖果不能满足任何孩子
j++
s++
直到i>g.length j>s.length

分析

使用jump保存结果

第一步应该往哪跳?
假设从第一个位置 第一步跳跃到dis[0]
那么从0到dis[0]之间的所有点都可能在第一次跳跃后达到

此时跳跃到的位置是pre
更新current为pre
同理 在到达pre之前 肯定发生了跳跃
直到扫描结束

用i从左向右扫描 使用一个变量current记录当前的子串
设置一个begin开始指向0位置
那么begin和i包含区域为当前连续子串current
i经过一次for自增1

当current中包含了[比较字符串]中的全部元素

对每次循环的current进行一次检测 若其长度最短 则保存到结果中

操作系统总结

操作系统的四大特性

并发

区分并发和并行

并行:指两个或多个事件在同一时刻同时发生
并发:指两个或多个事件在同一时间间隔内发生.这些时间宏观上是同时发生的,但微观上是交替发生的
操作系统的并发性指计算机系统中同时存在多个运行的程序
一个单核处理机CPU同一时刻智能执行一个程序,因此操作系统会负责协调多个程序交替执行(微观交替,宏观即用户角度就像在同时执行)
事实上,操作系统就是伴随着”多道程序技术”而出现的.因此,操作系统和程序并发是一起诞生的
当今计算机一般都是多核CPU 如Intel8代i3CPU是4核CPU,即同一时刻可以有4个程序并行执行,但依然需要操作系统的并发处理

共享

共享即资源共享,是指系统中的资源可供内存中多个并发执行的进程共同使用

共享方式
方式 说明 例子
互斥共享方式 系统中的某些资源,虽然可以提高给多个进程使用,但一个时间段内只允许一个进程访问该资源 使用QQ和微信视频时,同一时间摄像头只能分配给其中一个进程
同时共享方式 系统中的某些资源,允许一个时间段内有多个进程”同时”对它们进行访问 使用QQ发送文件A 同时微信发送文件B 宏观上看两个进程都在同时读取并发送,即两个进程都在访问硬盘资源.微观上,两个进程视交替访问硬盘的
所谓的”同时”往往是宏观的,而在微观上,这些进程可能是交替地对该资源进行访问

虚拟

虚拟是指把一个物理上的实体变为若干个逻辑上的对应物,物理实体是真实存在的,而逻辑对应物是用户感受到的
如单核CPU下运行了6个程序 让用户感受到6个CPU在运行

虚拟技术

虚拟技术 例子
空分复用技术 虚拟内存(物理内存<<需求内存 分配虚拟内存处理)
时分复用技术 虚拟处理器(单核处理器下运行多程序)

异步

异步是指在多道程序环境下 允许多个程序并发执行,但由于资源有限,进程的执行不是一贯到底的,而是走走停停,以不可预知的速度向前推进的,这就是进程的异步性

操作系统分类

中断

1、中断发生时,cpu以及进入核心态
2、中断发生后,当前运行的进程停止运行,并由操作系统内核对中断进行处理
3、对于不同的中断信号,会进行不同的处理

中断可以使CPU从用户态转换为核心态,使操作系统获得计算机的控制权
分类 特点
内中断 信号来源于CPU内部(如指令中断,硬件故障,软件中断)
外中断 信号来源于CPU外部(如人工干预,外设请求)

进程

具有动态性
进程是程序的一次执行过程,是系统进行资源分配和调度的一个独立单位
进程实体=程序段+数据段+PCB(进程控制块:存放操作系统所需的数据)

进程的状态

状态 特点
运行态 占有CPU,在CPU上运行
就绪态 具备运行条件,由于没有空闲CPU,暂时无法运行
阻塞态 因等待某一事件不能运行
创建态 进程正在被创建,操作系统为进程分配资源,初始化PCB
终止态 进程正在从系统中撤销,操作系统回收进程拥有的资源,撤销PCB

闭包

理解闭包

1、如何产生闭包?
    当一个嵌套的内部函数 引用了嵌套外部函数的局部变量时 产生闭包
2、闭包的理解
    *第一种:嵌套的内部函数
    *第二种:包含被引用变量(函数)的对象
3、产生闭包的条件
    *函数嵌套
    *内部函数引用了外部函数的数据(变量、函数)
    *执行外部函数
4、闭包的作用
    *使用函数内部的局部变量在函数执行完后,仍存活在内存中(延长局部变量的生命周期)
    *在函数外部可以操作(读写)函数内部的数据(变量/函数)
5、闭包的生命周期
    *产生:在嵌套内部函数定义执行完成时产生
    *死亡:在嵌套内部函数成为垃圾对象时
    *f = null

闭包的缺点与解决方案

缺点
    1、函数执行完后,函数内部的局部变量没有释放,占用内存时间会变长
    2、造成内存泄露
解决方案
    1、能不用就不用闭包
    2、及时释放

内存溢出与内存泄露

内存溢出
    内存溢出是一种程勋运行出现的错误
    当程序运行需要的内存超过了当前剩余内存是,跑出内存溢出错误
内存泄露
    占用的内存没有及时释放
    内存泄露积累过多容易产生内存溢出
    常见的内存泄露
        意外的全局变量
        没有及时清理的计时器或回调函数
        闭包
