关于vue里的MVVM

了解MVVM你需要知道

1、常见节点类型

节点 节点名称 nodeType
document 文档节点 9
element 元素节点 1
attribute 属性节点 2
text 文本节点 3

2、Object.defineProperty

键名 描述 默认值
configurable 是否可配置(重新定义) true
enumerable 是否可枚举(遍历查找) false
value 属性值(初始值) undefined
writable 是否可修改 false
get 回调函数:根据其他相关属性动态计算读取当前属性的值 undefined
set 回调函数:监视当前属性值的变化,更新其他相关的属性 undefined

例子

3、Object.keys()

返回对象自身可枚举属性组成的数组

4、obj.hasOwnProperty(prop)

判断prop是否是obj自身的属性

5、DocumentFragment

document VS documentFragment

node类型 更新与页面的关系
document 对应显示页面,包含n个element,一旦更新其中的某个元素界面都会更新
documentFragment 是内存中保存n个element的容器对象,不与界面关联,更新fragment中的element时界面不变

例:现有需求在 内存中将所有li的内容替换为updated data
解决方案:

数据代理

数据代理:通过一个对象代理对另一个对象中的属性进行操作(读/写)

模板解析

解析文本-大括号表达式

1、根据正则对象得到匹配的表达式字符串 子匹配(RegExp.$1)获取属性值
2、从data中取出表达式对应的属性值
3、将属性值设置为文本节点的textContent

解析指令-事件指令

1、从指令名中取出事件名
2、根据指令的值(表达式)从methods中得到对应的事件处理函数对象
3、给当前元素节点绑定事件名和回调函数的dom事件监听
4、指令解析完移除指令属性

数据劫持

1、修改属性 this.xxx = ‘xxx’
2、vm的set回调执行 (回调中修改的是data中的属性值)
3、触发data中属性的set回调执行
4、引起页面的修改

数据劫持涉及的类

1、observer:给data重新定义属性 添加set/get方法
在get中建立dep和watcher之间的关系,在set中监视key属性变化,通知所有相关订阅者(watcher)
2、dep:对应属性名 通知watcher
3、watcher:对应表达式 建立dep和watcher的关系 数据变化后回调函数调用 更新界面

流程图

双向数据绑定

1、前提:建立在单向数据绑定model->view之上
2、流程
在解析v-model指令时,给当前元素添加input事件
当input的value变化时,将最新的值赋值给当前表达式对应的data属性上(触发set回调进入步骤1)
