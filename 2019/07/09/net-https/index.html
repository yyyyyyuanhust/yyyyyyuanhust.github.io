HTTPS

HTTPS

https = http + ssl (secure socket layer)
端口443

解决问题

http的明文传输可能被拦截 引起重要信息泄露

加密手段

对称加密

常用算法:DES 3DES AES
原理:使用密钥s加密后必须使用s解密
优点:计算量小,加密解密速度较快,适合加密较大的数据
缺点:密钥的传输容易泄露 一个用户对应一个密钥 服务器管理麻烦

非对称加密

常用算法:RSA
原理:有一个公钥(public key)和一个私钥(private key),由加密算法生成.公钥加密只能私钥解密,私钥加密只能公钥解密
服务器保存了公钥和私钥,将公钥传递给客户端,客户端使用公钥加密后传递数据发送到服务器,服务器使用私钥解密
优点:加密解密使用不用钥匙,可以传输公钥,数据传输安全
缺点:计算量大,加密解密速度较慢

https加密过程

https结合了对称解密和非对称加密的优势,对数据进行加密和解密步骤如下
1、客户端对服务器发起https请求
2、服务器生成一个公钥和一个私钥,将SSL数字证书(包含公钥、服务器相关信息)发送给客户端
3、客户端解析SSL数字证书拿到公钥,验证服务器信息,客户端生成一个随机数使用公钥加密
4、客户端向服务器传输公钥加密后的随机数
5、服务器获取这个公钥加密后的随机数后用私钥解密
6、客户端使用随机数作为密钥对数据进行对称加密(此时服务器也知道密钥因此无需传输密钥)后传输
7、服务器获取密钥加密的数据 使用随机数密钥解密数据
